class mindee.input.page_options.PageOptions

PageOptions(page_indexes, operation, on_min_pages)

on_min_pages: int

Apply the operation only if document has at least this many pages.

Default: 0 (apply on all documents)

operation: str

Operation to apply on the document, given the page_indexes specified:

  • KEEP_ONLY - keep only the specified pages, and remove all others.

  • REMOVE - remove the specified pages, and keep all others.

page_indexes: Sequence[int]

Zero-based list of page indexes. A negative index can be used, indicating an offset from the end of the document.

[0, -1] represents the fist and last pages of the document.

parse(document_class, endpoint_name=None, account_name=None, include_words=False, close_file=True, page_options=None, cropper=False)

Call prediction API on the document and parse the results.

Parameters

  • document_class (TypeVar(TypeDocument, bound= Document)) – The document class to use. The response object will be instantiated based on this parameter.

  • endpoint_name (Optional[str], default: None) – For custom endpoints, the “API name” field in the “Settings” page of the API Builder. Do not set for standard (off the shelf) endpoints.

  • account_name (Optional[str], default: None) – For custom endpoints, your account or organization username on the API Builder. This is normally not required unless you have a custom endpoint which has the same name as standard (off the shelf) endpoint. Do not set for standard (off the shelf) endpoints.

  • include_words (bool, default: False) – Whether to include the full text for each page. This performs a full OCR operation on the server and will increase response time.

  • close_file (bool, default: True) – Whether to close() the file after parsing it. Set to False if you need to access the file after this operation.

  • page_options (Optional[PageOptions], default: None) – If set, remove pages from the document as specified. This is done before sending the file to the server and is useful to avoid page limitations.

  • cropper (bool, default: False) – Whether to include cropper results for each page. This performs a cropping operation on the server and will increase response time.

Return type
